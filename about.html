Project Monolith

Project Monolith was a tool written by me to automate several processes. We did not have access to the Airwatch API, so I wrote a webscraping tool that could imitate it.

Use Cases

A report of devices with old versions of applications was sent out daily, this tool could be given a specific application to redownload and attempt to fix them. It would send out a report when finished.

It was estimated this tool could do the same amount of work as 7 full time employees.

Libraries used

    selenium (web scraping)

    pandas (dataframes to organize web scraping)

    paramiko (used to ssh into linux box and ping devices)

    csv (used to generate the report)

    win32 (send the report in outlook to necessary audience)

    tkinter (asked for username and password upon startup)

    time (timer to see how long the process took)



Folder Comparison Tool

The folder comparison tool was really ground breaking for the company. It led to a 99.99% decrease in time spent on a task allowing engineers to focus on other tasks.

Before, our engineers got a folder of XML files and each one represented a profile. Whenever a change was made in the field, they had to compare the master folder of XML files to the proposed XML files. They used to do this by hand and review 60,000+ lines of code manually. I quickly realized this was a very bad approach and created a tool to solve the issue.

My tool reduced the time spent from 3 weeks to 3 minutes and allowed for greater accuracy of results.

Use Cases

Compares two folders for differences in files and generates a report on files that are not identical. Only works on text and XML files.

Libraries Used

difflib (generate the report)

os (retrieve the path to folders)

sys (read and write to files)

tkinter (ask user what folders to compare)

webbrowser (open up the webbrowser and display the report)
